# TDD 원칙

**구현 코드보다 테스트를 먼저 작성한다. 예외 없음.**

## 핵심 규칙

- 테스트가 없는 기능 구현은 시작하지 않는다.
- 실패하는 테스트(Red)를 먼저 작성하고, 통과시키는 최소 코드(Green)만 작성한다.
- Green 이후에만 리팩토링(Refactor)한다.
- 한 번에 하나의 테스트만 추가한다.

## 판단 기준

아래 중 하나라도 해당하면 구현 전에 테스트를 먼저 작성한다:

- 새 함수·메서드·클래스를 추가할 때
- 버그를 수정할 때 (버그를 재현하는 테스트 먼저)
- 기존 동작을 변경할 때

문서만 수정하거나 설정 파일만 바꾸는 경우는 예외.

## 루프

```
Red   → 실패하는 테스트 작성
Green → 테스트를 통과하는 최소 구현
Refactor → 동작을 유지하며 코드 정리
```

각 단계 사이에 테스트를 실행해 상태를 확인한다.
언어별 명령은 `/tdd` 스킬을 참조한다.
