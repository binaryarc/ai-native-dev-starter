# Agent Workflow

AI 코딩 에이전트와 일할 때의 기본 루프입니다.

## 1. Frame

작업을 시작하기 전에 문제를 좁힙니다.

```text
목표:
현재 증상:
관련 파일:
변경 가능 범위:
검증 방법:
```

## 2. Explore

에이전트가 코드를 읽고 기존 패턴을 파악하게 합니다.

요청 예시:

```text
먼저 관련 파일을 읽고 현재 구조를 요약해줘. 아직 코드는 수정하지 말고, 수정 계획과 리스크를 먼저 알려줘.
```

## 3. Implement

계획이 납득되면 구현을 맡깁니다.

```text
위 계획대로 구현해줘. 기존 스타일을 유지하고, 변경 범위를 관련 파일로 제한해줘.
```

## 4. Verify

검증은 워크플로의 일부입니다.

```text
가능한 테스트를 실행하고, 실패가 있으면 원인과 수정 여부를 구분해서 보고해줘.
```

## 5. Review

마지막에는 diff 중심으로 확인합니다.

```text
변경된 파일, 핵심 변경, 검증 결과, 남은 리스크를 짧게 정리해줘.
```

## Practical Rules

- 한 요청에는 한 가지 목적만 담습니다.
- 실패 로그는 원문 그대로 제공합니다.
- "알아서 좋게"보다 제약 조건을 명확히 씁니다.
- 에이전트가 만든 코드는 반드시 diff로 검토합니다.
