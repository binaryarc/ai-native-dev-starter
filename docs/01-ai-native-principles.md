# AI Native Principles

AI 네이티브 개발은 AI를 자동완성 도구가 아니라 협업 가능한 실행 주체로 다루는 방식입니다.

## 1. Specify The Outcome

에이전트에게 "무엇을 할지"보다 "어떤 상태가 되어야 하는지"를 먼저 줍니다.

좋은 요청은 다음을 포함합니다.

- 목표
- 현재 문제
- 허용되는 변경 범위
- 성공 기준
- 검증 방법

## 2. Keep Context Small And Useful

에이전트 성능은 컨텍스트 품질에 크게 좌우됩니다.

좋은 컨텍스트:

- 관련 파일 경로
- 실패 로그
- 재현 절차
- 기존 코드 패턴
- 하지 말아야 할 것

나쁜 컨텍스트:

- 관련 없는 전체 로그
- 추측성 요구사항
- 서로 충돌하는 지시
- 검증 불가능한 표현

## 3. Make Verification Explicit

에이전트에게 구현만 맡기면 품질은 흔들립니다. 테스트, 린트, 타입 체크, 수동 확인 기준을 명시해야 합니다.

예시:

```text
작업 후 npm test와 npm run lint를 실행하고, 실패하면 원인을 분석해서 수정해줘.
```

## 4. Prefer Small Batches

큰 작업을 한 번에 맡기면 리뷰 비용이 커집니다. 에이전트 작업은 작은 변경 단위로 쪼개는 편이 안전합니다.

권장 단위:

- 하나의 버그
- 하나의 화면
- 하나의 API 엔드포인트
- 하나의 리팩터링 목적

## 5. Humans Own Judgment

에이전트는 탐색, 초안, 반복 구현에 강합니다. 하지만 요구사항 해석, 제품 판단, 배포 리스크 판단은 사람이 책임져야 합니다.
