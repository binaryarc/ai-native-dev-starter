# Team Operating Model

팀에서 AI 코딩 에이전트를 쓰려면 도구보다 운영 규칙이 중요합니다.

## Shared Rules

팀 레포에는 다음 문서가 있어야 합니다.

- 에이전트용 작업 규칙
- 테스트 실행 방법
- 코드 스타일과 금지 패턴
- PR 작성 규칙
- 배포 전 체크리스트

## Ownership

AI가 만든 코드라도 책임자는 사람입니다.

권장 원칙:

- 에이전트 작업 단위마다 human owner를 둡니다.
- owner는 요구사항, diff, 검증 결과를 확인합니다.
- 에이전트가 만든 PR도 일반 PR과 같은 리뷰 기준을 적용합니다.

## Agent Friendly Repository

에이전트가 잘 일하는 레포는 사람이 봐도 좋습니다.

- 명확한 README
- 빠르게 실행되는 테스트
- 일관된 파일 구조
- 재현 가능한 개발 환경
- 의미 있는 에러 메시지

## Team Prompts

팀에서 반복적으로 쓰는 요청은 템플릿화합니다.

- 기능 구현 요청
- 버그 수정 요청
- 리팩터링 요청
- 코드 리뷰 요청
- CI 실패 분석 요청
